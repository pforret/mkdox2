## Lifecycle

1. **Proposed** — a member drafts a project page and brings it to a [meeting](../meetings/index.md).
2. **Approved** — committee agrees to take it on; logged in [Decisions](../decisions/index.md) if it commits resources.
3. **In progress** — the project lead reports at each meeting.
4. **Done** — final report; moved to the **Done** section above.

## Conventions

- One page per project.
- The page owns its action items — don't duplicate them across [meeting minutes](../meetings/index.md).
- Cross-link to the [topic](../topics/index.md) the project belongs to.
