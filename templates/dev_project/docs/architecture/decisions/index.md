# Architecture Decision Records

We capture significant architectural decisions as ADRs so future-us can recover the *why*.

## Conventions

- One file per decision, numbered: `NNNN-short-title.md`
- Status starts as **Proposed**, moves to **Accepted** when merged, becomes **Superseded** when a later ADR replaces it
- Use the [ADR template](template.md) as a starting point

## Process

1. Open a draft ADR as a PR.
2. Discuss in review — the PR is where the debate happens.
3. On merge, status becomes **Accepted** and the decision is binding until a later ADR supersedes it.
4. Append an entry to the table above.

> See [Workflow](../../contributing/workflow.md) for how ADRs interact with the PR process.
